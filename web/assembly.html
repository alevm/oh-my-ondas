Assembly Instructions

Based on verified hardware in hand — March 2026, Barcelona

Before You Start

Confirm these counts. If anything is short, note it before beginning.

  • Slide pots 45mm (P4272): ordered 5, confirm you have 5
  • Wire Kit 22AWG (P3174): check Adafruit box — not visible in photos
  • Helping Third Hand (P291): check Adafruit box — not visible in photos

Key Decisions

  1. Use the Teensy 4.1 built-in SD card (SanDisk Ultra 32GB already inserted). The Audio Shield’s SD slot goes unused — avoids SPI bus conflict with the LCD.
  2. LCD is a 2.8″ ILI9341 (P1770), not the originally spec’d 3.5″ HXD8357D. Better — ILI9341_t3 is the most optimized display library for Teensy with DMA support.
  3. Split I2C across two buses: Wire (18/19) for audio-path devices, Wire1 (16/17) for UI-polling devices.
  4. Wire the 4 main buttons directly to Teensy pins (not through MCP23017) for latency and interrupt capability.
  5. Power the ESP32-S2 from 5V through its own regulator, not from Teensy’s 3.3V.

Tools Needed

  • Soldering station (Diotronic 48W)
  • Solder wire 1mm
  • Flux paste
  • Breadboard ×2
  • Jumper wires (2 × 140pc)
  • Helping third hand (check if received)
  • Micro-USB cable for Teensy
  • Headphones with 3.5mm jack

1

Core Audio — Hear Sound

Goal: WAV file plays through headphones
Teensy 4.1Audio Shield Rev D2SD CardHeadphones

Step 1.1 — Prepare the Audio Shield

The Audio Shield has no headers soldered. Attach female headers to its underside so it mates with the Teensy’s pre-soldered male pins.

  1. Take 2× female 40-pin header strips from Diotronic.
  2. Snap or cut each to 14 pins (the Audio Shield uses 2 rows of 14).
  3. Place the female headers through the Audio Shield’s bottom-side holes.
  4. Stack the Audio Shield onto the Teensy to align everything — use the Teensy as a jig.
  5. Solder the female headers to the Audio Shield while sitting on the Teensy.
  6. Let cool. Separate the boards.

Note: Standard female headers are fine for breadboard prototyping — you’ll access all pins from the breadboard rows.

Step 1.2 — Prepare the SD Card

  1. Remove SD card from Teensy.
  2. Format as FAT32 (not exFAT).
  3. Create /samples/ folder on the root.
  4. Download PJRC test WAV files from the Audio data files page.
  5. Copy to SD card root (not inside /samples/).
  6. Re-insert into Teensy’s built-in slot (NOT the Audio Shield’s slot).

Step 1.3 — Stack and Connect

  1. Insert Teensy into breadboard #1 (spans the center gap).
  2. Stack Audio Shield on top via the soldered headers.
  3. Plug headphones into Audio Shield’s 3.5mm jack.
  4. Connect Teensy to computer via micro-USB.

Phase 1 — Teensy + Audio Shield stack on breadboard. No wires needed — all connections via stacking headers.

Step 1.4 — Flash and Test

  1. Select Board: Teensy 4.1, USB Type: Serial, CPU: 600MHz.
  2. Open: File → Examples → Audio → WavFilePlayer
  3. Set SD card pin:
#define SDCARD_CS_PIN    BUILTIN_SDCARD
// Comment out the Audio Shield SD lines:
// #define SDCARD_CS_PIN  10
// #define SDCARD_MOSI_PIN 7
// #define SDCARD_SCK_PIN  14

Upload. Open Serial Monitor at 9600 baud. You should hear WAV files through headphones.

If no sound: Check Serial Monitor for “Unable to access SD card” (format issue or wrong CS pin). Check headphone jack is fully inserted. Check Audio Shield is properly seated.

Phase 1 complete when: you hear audio in headphones.

Touch Sensing — Trigger Samples

Goal: Touching copper pads triggers audio samples
CAP1188Copper TapeJumper Wires

Step 2.1 — Wire CAP1188 to I2C1

The CAP1188 goes on the second I2C bus (Wire1), not the same bus as the Audio Shield.

  1. Solder header pins onto the CAP1188 breakout if not pre-soldered.
  2. Insert into breadboard.
  3. Wire: SDA → pin 17 (SDA1), SCL → pin 16 (SCL1), VIN → 3.3V, GND → GND
  4. Add I2C1 pull-up resistors (no built-in pull-ups on this bus): 10KΩ from pin 17 to 3.3V, 10KΩ from pin 16 to 3.3V. (2.2K would be better — grab from Diotronic next visit.)

Step 2.2 — Make Touch Pads

  1. Cut 8 pieces of copper tape, ~20mm × 20mm each.
  2. Stick onto cardboard or acrylic, spaced ~5mm apart in a 2×4 grid.
  3. Solder a short wire to each pad.
  4. Connect each wire to CAP1188 inputs C1–C8.

Phase 2 — CAP1188 on Wire1 (pins 16/17) with 10K pull-ups and 8 copper tape touch pads.

Step 2.3 — Test

Adafruit_CAP1188 cap = Adafruit_CAP1188();
// In setup():
Wire1.begin();
cap.begin(0x29, &Wire1);  // Address 0x29 on Wire1

Open Serial Monitor. Touch each pad — you should see touch events.

Step 2.4 — Integrate with Audio

Modify audio code: when pad N is touched, trigger sample N from SD card using AudioPlaySdWav objects.

Phase 2 complete when: touching copper pads triggers different audio samples.
3

Display — See What’s Happening

Goal: OLED shows status, LCD shows waveforms/UI
SSD1306 OLED 0.96″ILI9341 2.8″ TFTWS2812B NeoPixels

Step 3.1 — Wire OLED

  1. Insert OLED into breadboard.
  2. Wire: SDA → pin 18 (SDA0), SCL → pin 19 (SCL0), VCC → 3.3V, GND → GND
  3. Run I2C scanner on Wire — should show: 0x0A (SGTL5000) and 0x3C (SSD1306).
  4. Flash SSD1306 example. Display “Oh My Ondas” on screen.

Step 3.2 — Wire 2.8″ ILI9341 LCD (SPI)

PIN CONFLICT: Pin 8 is claimed by Audio Shield I2S. Use pin 15 for LCD RST instead.

Wiring:

  • CLK → pin 13 (SCK)
  • MOSI → pin 11
  • MISO → pin 12
  • CS → pin 10 (freed from Audio Shield SD)
  • D/C → pin 9
  • RST → pin 15
  • VIN → 3.3V, GND → GND
  • LITE → 3.3V (always on) or free PWM pin
#include <ILI9341_t3.h>
#define TFT_CS   10
#define TFT_DC   9
#define TFT_RST  15
ILI9341_t3 tft = ILI9341_t3(TFT_CS, TFT_DC, TFT_RST);

Step 3.3 — Wire NeoPixel Strip

  1. Data pin → pin 6
  2. VCC → 5V, GND → GND
  3. Add 470Ω resistor in series on data line.

If audio glitches: Switch to WS2812Serial library with a Serial TX pin instead of pin 6.

Phase 3 complete when: OLED shows text, LCD shows graphics, NeoPixels light up — audio still clean.
4

Analog Inputs — Faders Work

Goal: Slide pots control parameters, read by ADS1115
ADS1115Slide Pots ×5100nF Caps

Step 4.1 — Wire ADS1115

  • SDA → pin 18 (SDA0, shared bus)
  • SCL → pin 19 (SCL0)
  • VDD → 3.3V, GND → GND
  • ADDR → GND (address 0x48)

Step 4.2 — Wire Slide Pots

Each pot has 3 pins: left end, wiper (center), right end.

Pots 1–4: Left → GND, Right → 3.3V (NOT 5V), Wiper → ADS1115 A0–A3. Add 100nF cap from each wiper to GND.

Pot 5: Same wiring, wiper → Teensy pin 14 (A0) directly. Add 100nF cap.

Step 4.3 — Test

#include <Adafruit_ADS1X15.h>
Adafruit_ADS1115 ads;
void setup() {
  Serial.begin(115200);
  ads.begin(0x48, &Wire);
}
void loop() {
  Serial.print("A0: "); Serial.print(ads.readADC_SingleEnded(0));
  Serial.print("  A1: "); Serial.println(ads.readADC_SingleEnded(1));
  delay(100);
}

Values should sweep cleanly from near 0 to near 26000. If jumpy: check 100nF caps, check wiring isn’t near SPI lines.

Phase 4 complete when: all slide pots read smoothly with no jitter.
5

I/O Expansion — Encoders and Buttons

Goal: Rotary encoders and buttons respond via MCP23017
MCP23017 ×2DIP Sockets ×2EC11 Encoders ×7 Tactile Buttons ×45-Way Nav Switch

Step 5.1 — Wire Buttons Directly to Teensy

4 primary buttons go direct for lowest latency + interrupt capability:

  • MODE → pin 2
  • SHIFT → pin 3
  • REC → pin 4
  • PLAY → pin 5
pinMode(2, INPUT_PULLUP);  // MODE
pinMode(3, INPUT_PULLUP);  // SHIFT
pinMode(4, INPUT_PULLUP);  // REC
pinMode(5, INPUT_PULLUP);  // PLAY
// Pressed = LOW, Released = HIGH. Use Bounce2 for debouncing.

Step 5.2 — Install MCP23017s

  1. Press MCP23017 chips into DIP sockets. Watch the notch — pin 1 at notch end.
  2. Both on I2C1 (same bus as CAP1188):

MCP23017 #1: SDA (chip pin 13) → Teensy pin 17 (SDA1), SCL (chip pin 12) → pin 16 (SCL1), VDD → 3.3V, VSS → GND, A0/A1/A2 → GND (address 0x20), RESET → 3.3V

MCP23017 #2: Same wiring, A0 → 3.3V (address 0x21). Add 100nF cap across VDD–VSS on each.

Step 5.3 — Wire Encoders

Each EC11 has pins: A, B, C (common), switch. Connect C to GND, A/B/SW to MCP23017 GPIOs.

MCP #1 (16 GPIO): 5 encoders (15 pins: A, B, SW each). 1 spare.

MCP #2 (16 GPIO): 2 encoders (6 pins) + 5-way nav (5 pins) = 11 used, 5 spare.

Step 5.4 — Wire 5-Way Nav Switch

Common → GND. Up/Down/Left/Right/Select → MCP23017 #2 GPIOs. Enable internal pull-ups in firmware.

Step 5.5 — Test

Wire1.begin();
// I2C scanner on Wire1...
// Should see: 0x20 (MCP #1), 0x21 (MCP #2), 0x29 (CAP1188)

If devices don’t appear: check pull-ups on SDA1/SCL1, check RESET tied high, check address jumpers.

Phase 5 complete when: all 7 encoders rotate and click, nav switch responds in 5 directions, buttons work.

GPS — Location Stamps

Goal: GPS acquires fix, coordinates appear on display
PA1010D GPS (STEMMA QT)

Step 6.1 — Wire GPS

  • SDA → pin 18 (SDA0, shared with Audio Shield + OLED + ADS1115)
  • SCL → pin 19 (SCL0)
  • VIN → 3.3V, GND → GND

I2C scanner on Wire should now show: 0x0A (SGTL5000), 0x10 (PA1010D), 0x3C (OLED), 0x48 (ADS1115).

Step 6.2 — Test GPS

#include <Adafruit_GPS.h>
Adafruit_GPS GPS(&Wire);
void setup() {
  Serial.begin(115200);
  GPS.begin(0x10);
  GPS.sendCommand(PMTK_SET_NMEA_OUTPUT_RMCGGA);
  GPS.sendCommand(PMTK_SET_NMEA_UPDATE_1HZ);
}

Test outdoors or near a window. PA1010D built-in antenna needs sky view. First fix: 30–60 seconds.

Step 6.3 — Display GPS on OLED

display.setCursor(0, 0);
display.print("Lat: "); display.println(GPS.latitudeDegrees, 6);
display.print("Lon: "); display.println(GPS.longitudeDegrees, 6);
display.print("Sats: "); display.println(GPS.satellites);
Phase 6 complete when: GPS shows Barcelona coordinates on OLED.
7

WiFi Link — AI Connection

Goal: ESP32 connects to WiFi and communicates with Teensy via UART
SparkFun ESP32-S2 Thing Plus

Step 7.1 — Program ESP32 Separately

Flash a WiFi test via its own USB port before connecting to the Teensy.

#include <WiFi.h>
const char* ssid = "YOUR_WIFI_SSID";
const char* password = "YOUR_WIFI_PASSWORD";
void setup() {
  Serial.begin(115200);
  WiFi.begin(ssid, password);
  while (WiFi.status() != WL_CONNECTED) { delay(500); Serial.print("."); }
  Serial.println("\nConnected!");
  Serial.println(WiFi.localIP());
}

Step 7.2 — Wire ESP32 to Teensy

Power from 5V, NOT from Teensy’s 3.3V. Do NOT connect ESP32 3.3V to Teensy 3.3V — each board has its own regulator.

  • ESP32 5V/VBUS → Teensy VIN (shared 5V rail from USB)
  • ESP32 GND → Teensy GND
  • ESP32 TX → Teensy pin 0 (RX1)
  • ESP32 RX → Teensy pin 1 (TX1)

Step 7.3 — Test UART

// Teensy side:
void setup() {
  Serial.begin(115200);   // USB serial for debug
  Serial1.begin(115200);  // UART to ESP32
}
void loop() {
  if (Serial1.available()) {
    String msg = Serial1.readStringUntil('\n');
    Serial.print("From ESP32: ");
    Serial.println(msg);
  }
}
Phase 7 complete when: Teensy receives messages from ESP32 over UART, audio still clean.

I2C Bus Architecture

Dual I2C bus split — Wire (audio-path devices) and Wire1 (UI-polling devices, external pull-ups required).

Power Distribution

USB 5V powers Teensy (3.3V LDO for peripherals) and ESP32-S2 (own regulator). Total 3.3V external draw ~60mA typical.
